## Architecture

**TanStack Start** (SSR-enabled React framework) built on top of Vite + Vinxi, deployed to **Cloudflare Workers** via `wrangler.jsonc`.

### Routing

File-based routing via `@tanstack/react-router`. Routes live in `src/routes/`. **Never edit `src/routeTree.gen.ts`** — it is auto-generated by the TanStack Router Vite plugin on every save.

- `src/routes/__root.tsx` — root layout: sets `<html>/<head>/<body>`, global meta tags, injects the stylesheet, and defines the 404 and global error UI.
- `src/router.tsx` — creates the router instance with `routeTree`, scroll restoration, and the default error component.
- New routes: create a file in `src/routes/` using `createFileRoute`. Nested paths use directory nesting or dots (e.g. `blog.post.tsx` → `/blog/post`).

### Styling

**TailwindCSS v4** with the `@import "tailwindcss"` syntax. Design tokens are defined entirely in `src/styles.css` as CSS custom properties in `:root` / `.dark`, then mapped to Tailwind utilities in the `@theme inline` block. All color values use **oklch format**. To add a new semantic color: add it to `:root` and `.dark`, then register it under `@theme inline`.

Dark mode is toggled by adding the `.dark` class to a parent element (`@custom-variant dark (&:is(.dark *))`).

### UI Components

**shadcn/ui** (New York style) — all components are already installed in `src/components/ui/`. Use the `cn()` helper from `@/lib/utils` for conditional class merging (clsx + tailwind-merge).

Icon library: **lucide-react**.

### Path Aliases

`@/*` resolves to `src/*` (configured in `tsconfig.json` and handled automatically by the Vite config).

### Vite Config

`vite.config.ts` delegates entirely to `@lovable.dev/vite-tanstack-config`. Do **not** manually add plugins that this config already includes (TanStack Start, React, Tailwind, tsconfig paths, Cloudflare adapter, component tagger). Pass any extra Vite config through `defineConfig({ vite: { ... } })`.
